== ESPD-EDM v5.0.0 overview

* The upcoming release v5.0.0 was presented as a central part of the meeting. The list of the model changes was reviewed with details:
** Model implemented in Enterprise Architect (moved from Excel).
** Restructure of GitHub repositories.
** New code lists updated to align with those used in eForms (replacement of single “criterion” with two “exclusion-ground” and “selection criterion” code lists).
** Improvements related to validation (generation of validation scripts).
** Change of schema to UBL 2.4.

== Restructure of repositories

* ESPD-EDM team presented the restructure of GitHub repositories, from conceptual model to implementation. New approach ensures consistency, reduced duplication and makes updates easier. In short, we start from conceptual layer, move to supporting tools, and finally reach the implementation layer:
** The https://github.com/OP-TED/espd-conceptual-model[espd-conceptual-model] defines the “what” as the single source of truth.
** The https://github.com/OP-TED/espd-tools[espd-tools] provides reusable utilities.
** The https://github.com/OP-TED/ESPD-EDM[ESPD-EDM] is the implementation layer containing schemas, code lists, and artefacts in order to build the ESPD service.

== Validation improvements

* XSLT files created from schematron files to validate ESPD Request and Response forms. These are used in ITB testbed repository. Some of schematron files are hand-written, but most are generated from the code lists. All these files are kept in the repository https://github.com/OP-TED/espd-validation-schematron[https://github.com/OP-TED/espd-validation-schematron].
* New Unix/Bash Shell script validation-schematron.sh has been created using more modern Bash. The script could be run from a larger variety of operating systems.
* Updated schematron and XSLT files: aligned with code lists for v5.0.0, improved consistency of checks.
* Next steps:
** Automated generation of all schematron from code lists.
** Improvements to readability of code.
** Checks on the order of Exclusion Ground and Selection Criterion questions automatically generated from the data model.

== Next steps

* 5.0.0 release is planned for Q3 of 2026 with announced changes included: data model in JSON format (extracted from Enterprise Architect), new code lists, restructuring of GitHub repositories, improved generation of validation code, UBL 2.4.
* Long/mid term plan: revamp of “relied-upon” block as agreed with DG GROW, improved alignment with eProcurement Ontology and eForms, UBL elements review (for “Other criteria”).
* DG GROW commented on ongoing projects such as Business Wallet and public procurement directives.
