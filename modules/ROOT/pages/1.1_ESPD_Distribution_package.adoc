= ESPD Distribution Package

All the semantic assets needed to implement ESPD electronic documents are organised in a link:{url-tree}[distribution package] as follows: 

image:ESPD_Distribution_Package_v4.jpg[Distribution package,align="center",pdfwidth=50%,scaledwidth=50%]

_Figure 1. Distribution package_

[arabic]
. *criterion* folder contains the data structure of the ESPD taxonomy criterion files for the Request and the Response, as well as a sub-folder of the VBA scripts used to update the criterion files automatically for XML Path IDs generation. It also contains a sub-folder with a default ESPD Request and ESPD Response criterion files that do not illustrate multiple occurrences. 
. *xml-examples* folder contains UBL XML samples for the ESPD Request and ESPD Response that correspond to the criterion files provided, as well as the xslt files used for the transformation into UBL XML files. 
. *codelists* folder contains the data structure of the codelists used in ESPD and the related generic code files, as well as the xsl file used for the transformation into generic code files. 
. *ubl* folder contains the UBL distribution files that are currently used in the ESPD. 
. *conceptual-model* folder contains the UML conceptual model of the ESPD in .eap, .xmi and HTML format. Those formats are generated using Enterprise Architect. 
. *validation* folder contains the resulting files of the validation process based on Schematron rules and generated using ITB (Interoperability Testbed).
