= ESPD Architecture and Procurement Procedure Steps 

== ESPD Architecture Overview 

The ESPD-EDM goal is to produce the ESPD Request and the ESPD Response files that are going to be used to develop the ESPD Service. To produce those files there are key components that are used (UBL, UML, data structure, XSL(T), XML), with regards to the other related projects in the area of european procurement (EU Vocabulary, eCertis, etc.). 

Below is the figure of the general ESPD Architecture Overview. 

image:ESPD_Architecture_Overview.jpg[ESPD Architecture Overview,align="center",width=650,height=500]

_Figure 10. ESPD Architecture Overview_

The ESPD Architecture Overview is composed of various components that interact with each other to produce the final ESPD UBL XML Request and Response. 

* *The UBL (Universal Business Language) component* for the Request and the Response. It describes the language UBL that is used in the ESPD, in order to define xml elements and also attributes of xml elements. 

* *The UML component* for the Request and the Response. It describes the various elements and their attributes, the relations between elements, the related cardinalities. There is a BOV (Business Oriented View) model and a TOV (Technical Oriented View) model. The TOV uses UBL. 

* *The ESPD Criterion Data Structure component* for the Request and the Response. It describes, in an Excel file, for each criterion, its hierarchical tree structure of descriptive elements. The initial file is an Excel file that is transformed (saved) into an ODS (Open Document) file from which a first XML file is extracted, and then a ESPD UBL XML file is generated using XSLT transformation rules files. 

* *The ESPD UBL XML component* for the Request and the Response. It describes the resulting ESPD UBL XML files, for the Request (generated by the Buyer or CA -contracting authority-) and for the Response (generated by the EO -economic operator-), obtained by using XSLT transformations applied to the related Criterion ODS XML file. 

* *The ESPD Codelists Data Structure component*. It describes, in an Excel file, for each codelist used in the ESPD project, its attribute-value structure of codes. The initial file is an Excel file that is transformed (saved) into an XML Spreadsheet file, and then ESPD Codelists Generic Code files are generated using XSL transformation rules file. 

* *The ESPD Codelists generic code component*. It describes the resulting ESPD Codelists generic code files obtained by using XSL transformations applied to the related Codelists XML Spreadsheet file. 

* *The ESPD Validation component*. It validates the generated files produced (ESPD Request UBL XML, ESPD Response UBL XML, ESPD Codelists Generic Code) against UBL schema and also against a set of ESPD specific rules. 


== ESPD Procurement Procedure Steps 

The whole procurement procedure is a set of tasks that can be summarised by the following figures. 

The Buyer or the Contracting Authority (CA) initiates a procurement procedure.
