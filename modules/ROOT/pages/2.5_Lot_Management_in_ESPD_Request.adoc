= Lot Management in ESPD Request 

== ESPD version 3.0.0 

The ESPD version 3.0.0 includes a change regarding how lots are managed. This is the outcome of the alignment with eForms.

In the version 3.0.0, the ESPD have the following logic:

There will be 1 ESPD request stating the Exclusion Grounds Criteria and the Selection Criteria. The figure below shows the logic:

image:Overview_new_lot_management_approach.jpg[Overview of new lot management approach.,width=497,height=129]

_Figure 13. Overview of new lot management approach._

The ESPD request includes Exclusion Grounds that are general for all the Lots in which the procedure is divided in. However, for Selection Criteria it is different since in the procedure it is possible to have different Lots for different Selection Criteria. 

Therefore, in the ESPD Request the different Selection Criteria will have to include the references to the lots to which they apply. It will be done using the UBL 2.3 element *cac:ProcurementProjectLotReference* that is defined for the parent element *cac:TenderingCriterion*.

== ESPD version 4.0.0 

The ESPD version 4.0.0 includes additionnal changes regarding how lots are managed. 

In fact, there is also a business rule that requires to be able to define different lots for different requirements (REQUIREMENT_GROUP) in a given Selection Criteria. To address this need, the image below illustrates the solution provided and implemented. 

image:Lot_Management_in_ESPD_v4.jpg[Lot Management in ESPD 4.0.0,width=650,height=500]

_Figure 13_c. Lot Management in ESPD 4.0.0 

An extra-level of REQUIREMENT_GROUP of multiple cardinality has been added to all Selection Criteria. This level contains a REQUIREMENT element of multiple cardinality as well that is used to define a set of lots (datatype LOT_IDENTIFIER). The specific requirements bound to the set of lots appear right below in a REQUIREMENT_SUBGROUP element. That way, the duplication of the extra-level of REQUIREMENT_GROUP allows to define different specific requirements (REQUIREMENT_SUBGROUP) for different lots or set of lots (REQUIREMENT of datatype LOT_IDENTIFIER). 

Hence, the ESPD 3.0.0 allows different lots for different Selection Criteria, whereas the ESPD 4.0.0 allows in addition different lots (or set of lots) for different requirements.
